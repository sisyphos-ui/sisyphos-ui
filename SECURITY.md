# Security Policy

## Supported versions

Security fixes are published for the latest minor release line. Earlier versions only receive fixes if the vulnerability is severe and the fix is trivial.

## Reporting a vulnerability

Please **do not open a public GitHub issue** for security reports.

Instead, open a [private security advisory](https://github.com/sisyphos-ui/sisyphos-ui/security/advisories/new) on GitHub. Include:

- A short description of the vulnerability
- A proof of concept or reproduction steps
- The version(s) affected
- Any mitigations you're aware of

We aim to acknowledge every report within **72 hours** and to ship a fix within **14 days** of confirmation, depending on severity.

After a fix lands, we'll publish a GitHub security advisory with credit to the reporter (unless anonymity is requested).

Thank you for helping keep Sisyphos UI and its users safe.
